# 数据库设计文档

## 一、技术选型

| 项目 | 说明 |
|------|------|
| 数据库 | SQLite 3 |
| Node.js 驱动 | better-sqlite3 (v12.x) |
| 存储模式 | WAL (Write-Ahead Logging) |
| 数据库文件 | `backend/database/ecommerce.db` |

## 三、表结构设计

### 3.1 用户表 (users)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 用户 ID |
| username | TEXT | UNIQUE NOT NULL | 用户名 |
| password | TEXT | NOT NULL | 密码 |
| name | TEXT | | 昵称 |
| email | TEXT | | 邮箱 |
| avatar | TEXT | | 头像 URL |
| role | TEXT | CHECK('admin','user') DEFAULT 'user' | 角色 |
| created_at | TEXT | DEFAULT datetime('now','localtime') | 创建时间 |
| updated_at | TEXT | DEFAULT datetime('now','localtime') | 更新时间 |

### 3.2 商品表 (products)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 商品 ID |
| name | TEXT | NOT NULL | 商品名称 |
| price | REAL | NOT NULL CHECK(>=0) | 价格 |
| description | TEXT | | 描述 |
| category | TEXT | | 分类 |
| image | TEXT | | 图片 URL |
| stock | INTEGER | DEFAULT 0 CHECK(>=0) | 库存 |
| rating | REAL | DEFAULT 0 CHECK(0-5) | 平均评分 |
| sales | INTEGER | DEFAULT 0 | 销量 |
| status | INTEGER | DEFAULT 1 CHECK(0,1) | 状态: 1上架 0下架 |
| created_at | TEXT | | 创建时间 |
| updated_at | TEXT | | 更新时间 |

**索引:** `idx_products_category`, `idx_products_status`, `idx_products_name`

### 3.3 地址表 (addresses)

|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 地址 ID |
| user_id | INTEGER | FK → users(id) ON DELETE CASCADE | 用户 ID |
| receiver | TEXT | NOT NULL | 收货人 |
| phone | TEXT | NOT NULL | 手机号 |
| province | TEXT | NOT NULL | 省 |
| city | TEXT | NOT NULL | 市 |
| district | TEXT | NOT NULL | 区 |
| detail | TEXT | NOT NULL | 详细地址 |
| is_default | INTEGER | DEFAULT 0 CHECK(0,1) | 是否默认 |
| created_at | TEXT | | 创建时间 |
| updated_at | TEXT | | 更新时间 |

**索引:** `idx_addresses_user_id`

### 3.4 购物车表 (cart_items)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 购物车项 ID |
| user_id | INTEGER | FK → users(id) ON DELETE CASCADE | 用户 ID |
| product_id | INTEGER | FK → products(id) ON DELETE CASCADE | 商品 ID |
| quantity | INTEGER | DEFAULT 1 CHECK(>0) | 数量 |
| selected | INTEGER | DEFAULT 1 CHECK(0,1) | 是否选中 |
| created_at | TEXT | | 创建时间 |
| updated_at | TEXT | | 更新时间 |

**唯一约束:** `UNIQUE(user_id, product_id)`
**索引:** `idx_cart_items_user_id`

### 3.5 订单表 (orders)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 订单 ID |

| address_id | INTEGER | FK → addresses(id) | 收货地址 ID |
| remark | TEXT | | 备注 |
| paid_at | TEXT | | 支付时间 |
| shipped_at | TEXT | | 发货时间 |
| completed_at | TEXT | | 完成时间 |
| created_at | TEXT | | 创建时间 |
| updated_at | TEXT | | 更新时间 |

## 五、SQL 注入防护

本项目使用 `better-sqlite3` 的**预编译语句（Prepared Statements）**来防止 SQL 注入：

```js
// 正确：参数化查询（自动转义）
db.prepare('SELECT * FROM users WHERE username = ?').get(username);

// 禁止：字符串拼接
// db.prepare(`SELECT * FROM users WHERE username = '${username}'`).get();
```

所有模型层方法均使用 `?` 占位符传参，确保参数值被安全转义。

## 六、接口调用示例

### 6.1 用户登录

```bash
curl -X POST http://localhost:3000/api/user/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}'
```

响应:
```json
{
  "code": 200,
  "message": "登录成功",
  "token": "token-1-1751360400000",
  "user": { "id": 1, "username": "admin", "name": "管理员", ... }
}
```

### 6.2 获取商品列表

```bash
curl http://localhost:3000/api/products?page=1&pageSize=10&category=电子产品&sort=price
```

### 6.3 添加购物车

```bash
curl -X POST http://localhost:3000/api/cart \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer token-1-1751360400000" \
  -d '{"productId":1,"quantity":2}'
```

### 6.4 创建订单

```bash
curl -X POST http://localhost:3000/api/orders \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer token-1-1751360400000" \
  -d '{"addressId":1,"payMethod":"wechat","remark":"请尽快发货"}'
```

### 6.5 获取订单列表

```bash
curl "http://localhost:3000/api/orders?status=pending&page=1&pageSize=10" \
  -H "Authorization: Bearer token-1-1751360400000"
```

### 6.6 取消订单

```bash
curl -X PUT http://localhost:3000/api/orders/1/cancel \
  -H "Authorization: Bearer token-1-1751360400000"
```

### 6.7 地址管理

```bash
# 获取地址列表
curl http://localhost:3000/api/addresses \
  -H "Authorization: Bearer token-1-1751360400000"

# 添加地址
curl -X POST http://localhost:3000/api/addresses \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer token-1-1751360400000" \
  -d '{"receiver":"王五","phone":"13600136000","province":"上海市","city":"上海市","district":"浦东新区","detail":"张江路100号","isDefault":false}'
```

## 七、性能优化措施

| 优化项 | 实现方式 |
|--------|---------|
| WAL 模式 | `PRAGMA journal_mode = WAL` - 读写并发不阻塞 |
| 索引优化 | 为高频查询字段（category, status, user_id, order_no 等）建立索引 |
| 预编译语句 | 使用 `db.prepare()` 缓存执行计划，避免重复编译 |
| 分页查询 | 所有列表接口支持 `page` + `pageSize` 分页 |
| 事务处理 | 订单创建使用 `db.transaction()` 保证原子性 |
| 连接单例 | 全局唯一数据库实例，避免频繁打开/关闭 |
| Busy Timeout | `PRAGMA busy_timeout = 5000` 避免写锁冲突 |
